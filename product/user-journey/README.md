# User Journey Documentation

This directory contains detailed user journey maps for Forseti/AmISafe personas.

## Purpose

User journey maps help us:
- Understand how users discover, adopt, and retain with our product
- Identify friction points and drop-off stages
- Optimize onboarding and activation flows
- Design features that support natural user behavior
- Measure success at each stage of the journey

## Current Journeys

### [Sarah - Urban Commuter](./sarah-urban-commuter.md)
**Status**: 🟢 Live Beta Testing  
**Target Segment**: Young urban professionals (25-35) who walk/use public transit  
**Key Moments**:
- **Discovery**: Google search after feeling unsafe
- **Aha Moment**: Comparing route z-scores on interactive map
- **Conversion**: Background alerts provide peace of mind worth $4.99/month
- **Advocacy**: Refers friends after 2-3 months of usage

**Success Metrics**:
- Activation: ≥3 hexagon clicks in first session
- D7 Retention: ≥3 sessions in first week
- Conversion: Paid subscription after 7-day trial
- Advocacy: NPS 9-10, ≥1 referral in 3 months

## Journey Map Template

When creating new user journeys, include:

1. **Persona Overview**
   - Demographics, goals, pain points
   - Lifestyle and context

2. **Discovery Phase**
   - How they find the product
   - Initial impressions

3. **Activation Phase**
   - First experience (landing, aha moment, signup)
   - Onboarding flow

4. **Retention Phase**
   - Week 1, Week 2-4, Month 2-3 usage patterns
   - Habit formation triggers

5. **Advocacy Phase**
   - Referral behavior
   - Power user evolution

6. **Metrics & Touchpoints**
   - Key actions at each stage
   - Success indicators

7. **Failure Modes**
   - Potential drop-off points
   - Risk mitigation strategies
